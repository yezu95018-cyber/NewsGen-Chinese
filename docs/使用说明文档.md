# 使用说明文档

## 1. 项目简介

本项目是一个中文新闻标题与摘要生成系统，面向中文新闻正文输入，基于 `IDEA-CCNL/Randeng-T5-77M-MultiTask-Chinese` 预训练生成模型进行微调，实现新闻标题生成、新闻摘要生成、关键词提取、结果分析和历史记录管理。

系统采用 Text-to-Text 多任务学习方式，将标题生成和摘要生成统一为生成任务，并通过 `生成标题：`、`生成摘要：` 两类提示词区分任务类型。前端使用 Gradio Blocks 实现，适合课程汇报、演示视频和实验结果展示。

## 2. 运行环境

推荐环境如下：

- 操作系统：macOS / Linux / Windows 均可
- Python：3.9
- 运行方式：本地命令行启动 Gradio Web 服务
- 访问地址：`http://127.0.0.1:7860`

建议使用 conda 创建独立环境：

```bash
conda create -n newsgen python=3.9 -y
conda activate newsgen
```

## 3. 依赖库说明

主要依赖如下：

- `torch`：模型推理与张量计算。
- `transformers==4.26.1`：加载 Randeng-T5 相关模型与 tokenizer。
- `pytorch-lightning==1.4.9`：加载训练阶段保存的 Lightning checkpoint。
- `torchmetrics==0.5.0`：训练评估相关指标依赖。
- `sentencepiece`：T5 tokenizer 分词依赖。
- `jieba`：中文关键词提取。
- `rouge`、`nltk`：生成结果评测。
- `scikit-learn`、`numpy`、`pandas`：数据处理与实验辅助。
- `gradio`：Web 前端展示。

安装依赖：

```bash
pip install -r requirements.txt
```

## 4. 数据准备说明

GitHub 仓库不直接提交完整训练数据，只提供 `samples/` 目录下的小样例文件，便于查看数据格式。

本地完整数据目录约定如下：

```text
data/title_raw/
data/summary_raw/
data/title_processed/
data/summary_processed/
data/multitask_processed/
```

其中：

- `data/title_raw/`：标题生成原始数据。
- `data/summary_raw/`：摘要生成原始数据。
- `data/title_processed/`：标题生成处理后 JSONL 数据。
- `data/summary_processed/`：摘要生成处理后 JSONL 数据。
- `data/multitask_processed/`：标题与摘要合并后的多任务 JSONL 数据。

当前已生成 GitHub 展示样例：

- `samples/multitask_sample.json`
- `samples/title_sample.json`
- `samples/summary_sample.json`

## 5. 模型文件准备说明

训练好的 checkpoint 文件体积较大，不上传 GitHub。本地运行前需要将 `.ckpt` 文件放入：

```text
outputs/multitask_v2_10000/
```

系统启动时会自动查找该目录下的 `.ckpt` 文件。若目录不存在或没有 checkpoint，会提示找不到模型文件。

## 6. 启动方式

在项目根目录执行：

```bash
conda activate newsgen
python app.py
```

启动成功后浏览器访问：

```text
http://127.0.0.1:7860
```

如果端口被占用，可以先关闭已有 Gradio 进程，再重新运行。

## 7. 前端功能说明

系统前端包含三个 Tab：

- 新闻生成：输入新闻正文，选择摘要长度和关键词数量，一键生成标题、摘要、关键词和结果分析。
- 我的记录：查看当前用户名下的历史生成记录，刷新记录，查看最近一条详情，并导出 JSON。
- 系统说明：展示模型、训练方式、数据规模、验证指标和系统功能。

生成结果分析包括：

- 生成耗时
- 标题字数
- 摘要字数
- 标题关键词覆盖率
- 摘要关键词覆盖率

## 8. 用户登录说明

系统采用轻量级用户名登录，不涉及密码和权限认证。用户在顶部输入用户名并点击“登录”后，系统会将后续生成记录归属到该用户名。

切换用户时，只需输入新的用户名并再次点击“登录”。不同用户的历史记录会按照用户名进行区分。

## 9. 历史记录说明

历史记录默认保存到：

```text
data/user_history.json
```

该文件用于本地演示，不建议提交到 GitHub。`.gitignore` 已忽略该文件。

导出的用户记录会保存到：

```text
exports/
```

或系统配置的导出目录。课程提交时不要提交真实用户历史记录。

## 10. 常见问题与解决方法

### 10.1 找不到 checkpoint

确认 `.ckpt` 文件已放入：

```text
outputs/multitask_v2_10000/
```

### 10.2 Hugging Face 网络连接失败

如果基础模型文件已缓存，本地仍可加载；如果首次运行且本地没有缓存，需要联网下载 `IDEA-CCNL/Randeng-T5-77M-MultiTask-Chinese`。

### 10.3 7860 端口被占用

关闭已有服务后重新运行：

```bash
python app.py
```

### 10.4 页面能打开但生成较慢

CPU 推理会比 GPU 慢。可以减少输入正文长度，或在 GPU 环境下运行。

### 10.5 历史记录没有显示

请先输入用户名并登录，再进行生成或刷新记录。
