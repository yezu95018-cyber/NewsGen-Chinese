# 报告写作素材

## 项目背景

随着互联网新闻数量持续增长，用户需要在短时间内快速了解新闻事件的核心内容。新闻标题和摘要是新闻文本中最重要的信息压缩形式，标题通常用于概括主题，摘要则用于保留主要事实和背景信息。因此，中文新闻标题生成和摘要生成是自然语言处理领域中具有实际应用价值的文本生成任务。

传统文本生成方法通常需要针对不同任务分别建模，而预训练生成模型能够将多种自然语言处理任务统一表示为“文本输入到文本输出”的形式。本项目选择中文预训练生成模型 Randeng-T5 作为基础模型，将标题生成和摘要生成统一建模，并构建了一个可以直接交互演示的新闻生成系统。

## 项目目标

本项目的目标是实现一个面向中文新闻正文的标题与摘要生成系统。用户输入一段中文新闻正文后，系统能够自动生成新闻标题和新闻摘要，并进一步提取正文关键词，统计生成耗时、字数和关键词覆盖率。除了模型推理功能外，系统还需要具备较完整的课程展示能力，包括用户名登录、用户历史记录保存、历史记录查看、结果导出和系统说明页面。

项目不仅关注模型效果，也关注系统完整性。最终系统应能够支撑课程报告撰写、PPT 展示和演示视频录制，体现从数据处理、模型微调、模型推理到前端系统实现的完整流程。

## 技术路线

本项目采用 Randeng-T5 预训练生成模型作为基础模型。Randeng-T5 属于 Text-to-Text 模型，可以将不同任务统一为文本输入和文本输出。项目使用 `生成标题：` 和 `生成摘要：` 两类任务提示词，将新闻标题生成和新闻摘要生成统一到同一个模型中。

在数据处理阶段，标题生成数据和摘要生成数据分别被转换为统一 JSONL 格式，每条样本包含 `src` 和 `tgt` 字段。随后将两类任务样本混合，构建多任务训练集。模型训练完成后，前端通过 `NewsGenerator` 封装加载 checkpoint，并在 Gradio 页面中调用推理函数完成生成。

系统分析部分使用 jieba TF-IDF 方法提取关键词，并计算关键词在标题和摘要中的覆盖率。历史记录部分采用 JSON 文件进行本地存储，便于轻量级演示。

## 数据集说明

项目使用两类中文新闻数据。标题生成数据由新闻正文和新闻标题组成，摘要生成数据由新闻正文和人工摘要组成。预处理后，标题生成样本被转换为 `生成标题：新闻正文` 到 `新闻标题` 的形式，摘要生成样本被转换为 `生成摘要：新闻正文` 到 `新闻摘要` 的形式。

正式实验中，训练数据包含标题生成 5000 条和摘要生成 5000 条，验证数据包含标题生成 1000 条和摘要生成 1000 条。由于完整数据文件较大，GitHub 仓库不直接上传原始数据和处理后的大规模数据，只在 `samples/` 目录中提供少量样例数据，用于展示数据格式。

## 模型方法

本项目的基础模型为 `IDEA-CCNL/Randeng-T5-77M-MultiTask-Chinese`。模型训练采用多任务 Text-to-Text 方法，通过在输入前添加任务提示词，让同一个模型学习不同生成目标。输入为 `生成标题：` 时，模型生成新闻标题；输入为 `生成摘要：` 时，模型生成新闻摘要。

推理阶段，标题生成设置较短的最大生成长度，并进行简单后处理裁剪，使标题更符合新闻标题简洁明确的特点。摘要生成则根据用户选择的“短”“中”“长”控制生成长度。模型使用 beam search 生成结果，并通过重复 n-gram 约束减少重复表达。

需要说明的是，当前系统中的关键词主要用于结果分析，并未直接输入模型参与生成。后续可以将关键词加入提示词，扩展为关键词增强生成方法。

## 系统功能

系统主要功能包括新闻标题生成、新闻摘要生成、关键词提取、摘要长度控制、关键词覆盖率分析、生成耗时统计、用户登录与切换用户、我的生成记录、历史记录导出等。

用户进入系统后，先输入用户名进行轻量级登录。随后可以在“新闻生成”页面输入新闻正文，选择摘要长度和关键词数量，点击按钮后系统会生成标题、摘要、关键词和结果分析。生成结果会自动保存到当前用户的历史记录中。用户可以在“我的记录”页面查看历史生成记录和最近一条详情，也可以导出当前用户记录为 JSON 文件。

## 前端实现

前端系统使用 Gradio Blocks 实现，页面结构清晰，包含顶部项目介绍和三个主要 Tab：“新闻生成”“我的记录”“系统说明”。“新闻生成”页面采用左右两栏布局，左侧为输入与参数控制，右侧为生成结果展示。“我的记录”页面用于展示历史记录和导出结果。“系统说明”页面用于展示模型、训练数据、验证指标和系统功能说明。

与简单 demo 相比，本项目的前端不仅提供模型输入输出，还集成了用户状态、历史记录、结果分析和课程展示信息，更适合课程汇报截图和演示视频。

## 实验结果

正式训练使用标题生成 5000 条、摘要生成 5000 条作为训练数据，使用标题生成 1000 条、摘要生成 1000 条作为验证数据，训练 1 epoch。模型 checkpoint 文件大小约 296M。

验证指标为 BLEU = 0.1602，ROUGE = 0.2837，ROUGE-1 = 0.3458，ROUGE-2 = 0.2290，ROUGE-L = 0.3074。从指标看，模型已经能够学习新闻标题和摘要的基本生成模式，但由于训练数据规模和训练轮数有限，生成质量仍有进一步提升空间。

## 结果分析

从前端测试结果看，模型生成的标题通常较短，具有一定抽取式特征，能够从正文中提炼出主题词或关键短语。摘要生成结果相对更长，能够覆盖新闻事件的主要内容。关键词覆盖率可以辅助观察生成结果是否包含正文核心词，但不能完全代表生成质量，因为有些语义相关表达并不一定包含完全相同的关键词。

系统在推理阶段对标题加入长度约束和后处理裁剪，主要是为了避免标题过长或混入摘要式表达。这一处理符合新闻标题简洁性的要求，也能提升前端展示效果。

## 不足与改进方向

当前项目仍存在一些不足。首先，训练数据规模相对有限，训练轮数只有 1 epoch，模型生成质量还有提升空间。其次，关键词目前只用于结果分析，没有直接参与生成，后续可以将关键词加入输入提示词，实现关键词增强标题和摘要生成。第三，历史记录目前使用 JSON 文件保存，适合轻量级演示，但如果扩展为多人长期使用系统，可以改为 SQLite 或其他数据库。最后，当前评价主要依赖 BLEU 和 ROUGE，后续可以加入人工评价、事实一致性分析和更多生成质量指标。
