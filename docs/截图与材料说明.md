# 截图与材料说明

## 需要截图的页面

建议最终至少准备以下截图：

1. 系统首页截图：展示项目标题、简介、用户名登录框和三个 Tab。
2. 新闻生成输入截图：展示新闻正文输入框、摘要长度选择、关键词数量滑块和示例新闻。
3. 新闻生成结果截图：展示生成标题、生成摘要、关键词和结果分析。
4. 关键词与覆盖率截图：重点截取关键词、生成耗时、标题字数、摘要字数和关键词覆盖率。
5. 我的记录页面截图：展示历史记录表格和最近一条详情。
6. 导出记录截图：展示导出 JSON 后的状态或生成的导出文件。
7. 系统说明页面截图：展示基础模型、训练数据、验证数据和验证指标。
8. 终端启动截图：展示 `python app.py` 启动并出现 `http://127.0.0.1:7860`。
9. checkpoint 或训练日志截图：展示 `outputs/multitask_v2_10000/` 中 checkpoint 文件或训练日志指标。

## 每张截图用于报告哪一部分

- 系统首页截图：用于“系统设计”或“运行界面展示说明”章节。
- 新闻生成输入截图：用于“系统实现”章节，说明用户输入和参数设置。
- 新闻生成结果截图：用于“测试结果与分析”章节，展示模型输出。
- 关键词与覆盖率截图：用于“结果分析”章节，说明关键词覆盖率和耗时统计。
- 我的记录页面截图：用于“系统功能”章节，说明历史记录管理。
- 导出记录截图：用于“系统实现”或“测试过程”章节，说明导出功能可用。
- 系统说明页面截图：用于“模型方法”或“实验设置”章节，说明模型和指标。
- 终端启动截图：用于“运行环境与启动方式”章节。
- checkpoint 或训练日志截图：用于“数据集与实验设置”或“实验结果”章节。

## PPT 中建议放哪些截图

PPT 中不需要放太多细节图，建议选择最能体现系统完整性的截图：

- 第 7 页“前端界面展示”：放系统首页、新闻生成页面和我的记录页面。
- 第 8 页“实验结果与指标”：放系统说明页面中的指标表或训练日志截图。
- 第 9 页“测试结果分析”：放标题摘要生成结果和关键词覆盖率分析截图。

如果 PPT 页数有限，可以将三张界面截图拼成一页，并在旁边用简短文字说明功能。

## 演示视频建议展示哪些功能

演示视频建议按实际操作顺序展示：

1. 终端运行 `python app.py`，打开浏览器页面。
2. 输入用户名并登录，说明支持切换用户。
3. 在“新闻生成”页面选择或粘贴新闻正文。
4. 调整摘要长度和关键词数量。
5. 点击“一键生成标题与摘要”，等待输出。
6. 展示生成标题、生成摘要、关键词和结果分析。
7. 切换到“我的记录”，刷新并查看最近一条详情。
8. 点击导出 JSON，展示导出结果。
9. 切换到“系统说明”，介绍模型、数据和指标。

视频讲解时建议强调：本项目不是直接调用大模型 API，而是基于开源预训练模型微调；关键词目前用于结果分析，后续可以扩展为关键词增强生成；历史记录功能体现了系统实现和课程展示的完整性。
