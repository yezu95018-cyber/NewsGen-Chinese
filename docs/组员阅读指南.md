# 组员阅读指南

## 项目一句话概括

本项目是一个基于 Randeng-T5 预训练生成模型微调的中文新闻标题与摘要生成系统，支持新闻生成、关键词分析、生成耗时统计、用户历史记录和课程展示前端。

## 当前项目已完成功能

目前项目已经完成了从数据处理、模型训练到前端展示的主要流程。模型部分基于 `IDEA-CCNL/Randeng-T5-77M-MultiTask-Chinese`，将标题生成与摘要生成统一建模为 Text-to-Text 多任务学习，通过 `生成标题：` 和 `生成摘要：` 两类提示词区分任务。前端部分使用 Gradio Blocks 实现，包含“新闻生成”“我的记录”“系统说明”三个页面。

系统功能包括新闻标题生成、新闻摘要生成、摘要长度选择、关键词提取、标题关键词覆盖率、摘要关键词覆盖率、生成耗时统计、轻量级用户名登录、用户历史记录保存、我的记录查看和当前用户记录导出。

## 写报告时重点阅读哪些文件

如果另一位组员主要负责报告撰写，建议优先阅读以下文件：

- `README.md`：项目总体介绍、功能、技术路线、环境运行和项目结构。
- `docs/项目报告.md`：已经整理好的报告初稿，可继续补充截图和测试结果。
- `docs/报告写作素材.md`：可直接改写进正式报告的段落素材。
- `docs/实验结果汇总.md`：模型配置、checkpoint、指标解释和测试样例。
- `docs/截图与材料说明.md`：报告、PPT 和演示视频需要补充的截图说明。
- `docs/演示PPT大纲.md`：PPT 页码结构和每页重点。
- `docs/演示视频脚本.md`：录制演示视频时可直接参考的讲解顺序。
- `app.py`：前端系统入口，能看到三个 Tab 和功能交互。
- `scripts/inference_utils.py`：推理封装，包含模型调用、关键词提取、覆盖率和耗时统计。

## 不能上传 GitHub 的文件

以下内容不要提交到 GitHub：

- `outputs/`：训练输出目录，包含 checkpoint。
- `*.ckpt`、`*.pt`、`*.pth`、`*.bin`、`*.safetensors`：模型权重文件。
- `data/title_raw/`：标题原始大数据。
- `data/summary_raw/`：摘要原始大数据。
- `data/title_processed/`、`data/summary_processed/`、`data/multitask_processed/`：处理后的大规模训练数据。
- `data/user_history.json`：本地用户真实历史记录。
- `exports/`：系统导出的用户记录。
- `logs/`：训练日志目录。

这些目录和文件已经在 `.gitignore` 中配置忽略。提交时不要使用 `git add .`，建议按 README 或提交清单中的命令选择性添加文件。

## 项目演示流程

推荐演示流程如下：

1. 打开终端，运行 `conda activate newsgen` 和 `python app.py`。
2. 浏览器访问 `http://127.0.0.1:7860`。
3. 在顶部输入用户名并登录，说明系统会按用户名保存历史记录。
4. 在“新闻生成”页面使用默认示例或粘贴新闻正文。
5. 选择摘要长度和关键词数量，点击“一键生成标题与摘要”。
6. 展示生成标题、生成摘要、关键词和结果分析。
7. 切换到“我的记录”，刷新并查看最近一条详情。
8. 演示导出当前用户记录为 JSON。
9. 切换到“系统说明”，讲解基础模型、训练数据规模和验证指标。

## 两人分工建议

可以按以下方式分工，最终根据实际贡献灵活调整：

- 成员 A：模型训练、数据处理、前端系统实现、实验记录整理。
- 成员 B：项目报告撰写、PPT 制作、演示视频录制、截图整理。

如果需要进一步细分，成员 A 可以负责解释模型方法、训练流程和代码结构；成员 B 可以负责把功能截图、实验结果和演示流程整理成报告与 PPT。最终提交前两位成员应共同检查 README、报告、PPT、视频和 GitHub 仓库，确保大文件和真实历史记录没有被误提交。
