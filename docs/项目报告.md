# 项目报告

## 1. 摘要

本项目围绕中文新闻文本生成任务，设计并实现了一个基于预训练生成模型的中文新闻标题与摘要生成系统。系统以 `IDEA-CCNL/Randeng-T5-77M-MultiTask-Chinese` 为基础模型，将新闻标题生成与新闻摘要生成统一建模为 Text-to-Text 任务，并通过 `生成标题：` 和 `生成摘要：` 两类任务提示词实现多任务学习。除模型推理外，系统还集成了关键词提取、关键词覆盖率统计、生成耗时统计、用户名登录、历史记录查看与 JSON 导出等功能，形成了较完整的课程展示系统。

本项目不是简单调用大模型 API，而是在开源预训练模型基础上进行数据预处理、模型微调、评测和系统集成。实验结果表明，经过小规模多任务微调后，模型能够根据中文新闻正文生成较为简洁的标题和摘要，并能通过前端界面完成可视化展示和结果分析。

## 2. 关键词

中文新闻生成；标题生成；摘要生成；Randeng-T5；Text-to-Text；多任务学习；Gradio

## 3. 引言

随着互联网新闻数量快速增长，用户在短时间内获取关键信息的需求不断增加。新闻标题和摘要能够帮助用户快速理解文本主题，是自然语言处理中的重要生成任务。传统方法往往将标题生成和摘要生成分别建模，而预训练生成模型可以将不同任务统一为文本到文本的生成形式，为多任务学习提供了便利。

本项目以中文新闻正文为输入，目标是自动生成新闻标题和新闻摘要，并通过关键词和覆盖率指标辅助分析生成结果。项目同时关注课程展示需求，构建了包含登录、历史记录和系统说明页面的 Gradio 前端。

## 4. 相关技术

### 4.1 预训练生成模型

预训练生成模型通过大规模语料学习通用语言表示和文本生成能力。Randeng-T5 是中文 Text-to-Text 生成模型，能够将分类、摘要、生成等任务统一为输入文本到输出文本的形式。

### 4.2 Text-to-Text 建模

Text-to-Text 方法将所有任务都表示为文本输入和文本输出。本项目中，标题生成样本表示为：

```json
{"src": "生成标题：新闻正文", "tgt": "新闻标题"}
```

摘要生成样本表示为：

```json
{"src": "生成摘要：新闻正文", "tgt": "新闻摘要"}
```

### 4.3 多任务学习

多任务学习通过共享模型参数，使模型同时学习多个相关任务。本项目将标题生成数据和摘要生成数据混合训练，使同一个 checkpoint 可以根据任务提示词完成不同生成任务。

### 4.4 关键词提取与覆盖率分析

系统使用 jieba TF-IDF 方法从输入新闻正文中提取关键词。关键词目前主要用于结果分析，未直接参与生成过程。系统统计生成标题和摘要中出现关键词的比例，用于辅助观察生成文本是否覆盖正文核心信息。

## 5. 系统设计

系统整体分为数据层、模型层、推理层和前端展示层。

- 数据层：包含标题生成数据、摘要生成数据和多任务混合数据。
- 模型层：基于 Randeng-T5 进行多任务微调。
- 推理层：封装标题生成、摘要生成、关键词提取和指标统计。
- 前端层：使用 Gradio Blocks 构建系统界面，包含新闻生成、我的记录和系统说明三个 Tab。

前端系统功能包括：

- 用户名登录与切换用户。
- 新闻正文输入与示例新闻填充。
- 摘要长度选择和关键词数量选择。
- 标题、摘要、关键词和分析结果展示。
- 历史记录保存、刷新、详情查看和导出。

## 6. 模型方法

基础模型为 `IDEA-CCNL/Randeng-T5-77M-MultiTask-Chinese`。模型输入由任务提示词和新闻正文拼接而成：

```text
生成标题：新闻正文
生成摘要：新闻正文
```

标题生成使用较短的最大生成长度，以得到简洁标题；摘要生成根据用户选择的“短”“中”“长”控制最大生成长度。推理阶段使用 beam search，并通过 `no_repeat_ngram_size` 减少重复片段。

多任务训练时，标题生成 5000 条样本和摘要生成 5000 条样本合并为训练集，标题生成 1000 条样本和摘要生成 1000 条样本合并为验证集。模型训练 1 epoch 后保存 checkpoint，用于前端加载和推理。

## 7. 数据集与实验设置

### 7.1 数据集

标题生成数据采用新闻正文与标题配对样本；摘要生成数据采用中文新闻正文与人工摘要配对样本。预处理后均转换为 JSONL 格式，每行包含 `src` 和 `tgt` 字段。

### 7.2 数据规模

| 数据类型 | 训练集 | 验证集 |
| --- | ---: | ---: |
| 标题生成 | 5000 | 1000 |
| 摘要生成 | 5000 | 1000 |
| 多任务合计 | 10000 | 2000 |

### 7.3 实验设置

- 基础模型：`IDEA-CCNL/Randeng-T5-77M-MultiTask-Chinese`
- 训练轮数：1 epoch
- 最大输入长度：512
- 最大输出长度：128
- 学习率：5e-5
- 模型文件大小：296M

### 7.4 验证指标

| 指标 | 数值 |
| --- | ---: |
| BLEU | 0.1602 |
| ROUGE | 0.2837 |
| ROUGE-1 | 0.3458 |
| ROUGE-2 | 0.2290 |
| ROUGE-L | 0.3074 |

## 8. 系统实现

系统前端入口为 `app.py`，推理逻辑封装在 `scripts/inference_utils.py` 中。`NewsGenerator` 负责加载 checkpoint、调用模型生成标题和摘要，并返回关键词、生成耗时、字数和关键词覆盖率等信息。

前端使用 Gradio Blocks 构建，包含三个主要页面：

- 新闻生成：提供新闻正文输入框、摘要长度下拉框、关键词数量滑块和生成按钮。
- 我的记录：展示当前用户历史记录，支持刷新最近记录和导出 JSON。
- 系统说明：展示模型信息、训练信息、验证指标和系统功能说明。

历史记录以 JSON 文件形式保存在 `data/user_history.json`，记录内容包括用户名、生成时间、输入正文、标题、摘要、关键词和分析指标。该文件仅用于本地演示，不提交 GitHub。

## 9. 测试结果与分析

系统测试主要包括启动测试、生成测试、记录测试和导出测试。

### 9.1 启动测试

在 conda 环境中执行：

```bash
python app.py
```

浏览器访问 `http://127.0.0.1:7860`，系统页面可以正常打开。

### 9.2 生成测试

输入一段教育科技类新闻正文后，系统可以生成标题和摘要，并显示关键词。标题通常较短，能够概括新闻主题；摘要能够保留主要事件和背景信息。

### 9.3 结果分析

系统会展示生成耗时、标题字数、摘要字数、标题关键词覆盖率和摘要关键词覆盖率。关键词覆盖率可用于观察生成文本是否包含正文中的核心词，但该指标不能完全代表语义质量，需要结合人工阅读进行判断。

### 9.4 历史记录测试

登录用户名后生成新闻，系统会将记录保存到本地 JSON 文件。“我的记录”页面可以刷新记录并展示最近一条详情。导出功能可以生成当前用户的 JSON 文件，便于课程演示和结果留存。

## 11. 总结与展望

本项目完成了一个中文新闻标题与摘要生成系统，实现了基于 Randeng-T5 的多任务生成模型微调和 Gradio 前端展示。系统不仅能够生成标题和摘要，还提供关键词提取、覆盖率分析、耗时统计和用户历史记录管理，体现了课程项目所需的系统完整性和功能扩展。

目前系统仍有改进空间。关键词目前仅用于结果分析，尚未直接参与生成。后续可以尝试关键词增强生成，将关键词作为额外输入加入提示词；也可以使用 SQLite 或其他数据库替代 JSON 文件，提高历史记录管理能力；此外，还可以扩大训练数据规模、延长训练轮数、引入更多自动评价指标和人工评价，以进一步提升生成质量。
